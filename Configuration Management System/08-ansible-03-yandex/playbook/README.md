```markdown
# Ansible Playbook для установки NGINX, Lighthouse, Clickhouse и Vector

## Описание
Этот Ansible playbook предназначен для автоматизации процессов установки и настройки NGINX, Lighthouse, Clickhouse и Vector. Он работает с ключевыми компонентами инфраструктуры, позволяя быстро и эффективно развертывать необходимые сервисы на целевых хостах.

### Структура playbook
Playbook разбит на несколько частей, каждая из которых отвечает за установку и настройку определенного компонента:

1. **NGINX**:
   - Устанавливает NGINX и создает конфигурацию для него.

2. **Lighthouse**:
   - Устанавливает зависимости, создает директорию для Lighthouse, клонирует репозиторий и настраивает права доступа.

3. **Clickhouse**:
   - Устанавливает Clickhouse, загружает необходимые пакеты, создает базу данных.

4. **Vector**:
   - Устанавливает Vector, создает конфигурационный файл и системный юнит.

## Параметры
Playbook использует несколько переменных, которые могут быть определены в инвентарном файле или в окружении:

- `lighthouse_location_dir`: Директория для установки Lighthouse.
- `lighthouse_vcs`: URL-адрес репозитория Lighthouse.
- `clickhouse_version`: Версия Clickhouse, которая будет установлена.
- `clickhouse_packages`: Список пакетов Clickhouse, который нужно загрузить.
- `vector_url`: URL-адрес RPM-пакета Vector.

## Теги
Playbook включает следующие теги, которые можно использовать для выборочной установки и настройки компонентов:

- `nginx`: Все задачи, связанные с установкой и настройкой NGINX.
- `lighthouse`: Все задачи, относящиеся к установке Lighthouse.
- `clickhouse`: Все задачи, связанные с установкой и настройкой Clickhouse.
- `vector`: Все задачи, относящиеся к установке Vector.

### Пример использования
Для выполнения playbook с определенным тегом, можно использовать команду:
```

```console
ansible-playbook playbook.yaml --tags nginx
```
